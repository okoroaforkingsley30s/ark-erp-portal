ARK ONE 1.0.16 — Android Mobile Platform Repair

This cumulative repair provides:

1. Company launcher, round and adaptive icons generated from public/logo.png.
2. Native Back-button navigation; Dashboard/Login minimizes instead of exiting.
3. Android Go and 1 GB RAM performance profile.
4. Narrow-screen and safe-area compatibility.
5. Firebase Cloud Messaging token registration and sleeping-phone delivery.
6. Notification click routing into the correct ARK ONE screen.
7. User-selectable ARK ONE Bell, Gentle Chime and Urgent Alert channels.
8. Foreground GPS permission, opt-in activation and current-location sync.
9. Database-backed device registry and audited push outbox.
10. Existing minute notification worker integration for mobile delivery.

Security:

- google-services.json contains Firebase client identifiers.
- The Firebase Admin private key is NOT included.
- FIREBASE_SERVICE_ACCOUNT_BASE64 must remain a Supabase production secret.
- GPS is opt-in and does not perform undisclosed background tracking.

Install:

1. Extract this ZIP into the ARK ONE repository root with overwrite enabled.
2. Run:

   powershell -NoProfile -ExecutionPolicy Bypass -File .\Install-ARK-ONE-Android-Mobile-Repair.ps1

3. Review and commit the resulting source changes on a dedicated branch.
4. Merge only after all six GitHub checks pass.
5. Back up production, then run:

   npx supabase db push --linked
   npx supabase functions deploy dispatch-mobile-push --project-ref fryidzyhqhdenghyxjfp --no-verify-jwt
   npx supabase functions deploy dispatch-notification-outbox --project-ref fryidzyhqhdenghyxjfp --no-verify-jwt

6. Build a new APK only after production deployment succeeds.

Production verification:

1. Install the new APK, open it and sign in once.
2. Accept the notification permission if Android asks.
3. In ARK ONE Notifications, select a sound and enable GPS if desired.
4. Confirm the phone registration in Supabase SQL Editor:

   select user_email, platform, sound_key, active, last_seen_at,
          location_recorded_at
   from public.ark_mobile_devices
   order by last_seen_at desc;

5. Create a major ARK ONE notification for the signed-in user, lock the phone
   and allow up to one minute for the existing notification schedule to run.

The installer also builds:

   android\app\build\outputs\apk\debug\app-debug.apk

The Itel S15 must have Google Play services available for Firebase sleeping-phone
delivery. Android battery optimization may delay alerts if the user force-stops
ARK ONE; normal background and sleeping-phone delivery remains supported.
